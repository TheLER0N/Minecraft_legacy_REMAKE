# Пояснительная записка и Техническое задание (ТЗ)
## Разработка кроссплатформенного воксельного движка на базе Vulkan API

**Проект:** Minecraft Legacy Remake (Полный клон Legacy Edition с переходом к современным версиям)  
**Автор:** Таболин Марк  
**Стек:** C++20, Vulkan 1.3, SDL3, GLM, stb_vorbis.

---

### 1. Введение и концепция
Данная работа посвящена проектированию и реализации высокопроизводительного игрового движка для воксельных миров. Основная цель проекта — создание **полнофункционального клона Minecraft Legacy Console Edition**, в точности воссоздающего дух и механики консольных версий (PS3, PS4, Xbox 360, Xbox one). 

При этом архитектура движка проектируется с заделом на будущее: проект не ограничится контентом старых версий, а будет планомерно развиваться до **функционала современных версий Minecraft** (поддержка новых биомов, механик и сущностей), сохраняя при этом классический "Legacy" интерфейс и удобство управления.

Проект ориентирован на низкоуровневый контроль графического конвейера через API Vulkan и обеспечение идентичного игрового опыта на десктопных (Windows/Linux) и мобильных (Android) платформах.

### 2. Техническая архитектура движка
Движок построен по модульному принципу, разделяя ответственность между четырьмя ключевыми подсистемами:

*   **Platform Layer (SDL3):** Абстрагирует работу с окнами, событиями ОС и вводом. Использование SDL3 позволяет унифицировать обработку сенсорного экрана Android и геймпада на ПК.
*   **Render Core (Vulkan):** Реализует графический конвейер. Включает в себя управление Swapchain, создание шейдерных модулей (GLSL -> SPIR-V), работу с пулами дескрипторов и динамическое обновление буферов вершин для чанков.
*   **Game Logic & World Engine:** Отвечает за хранение данных мира, физику игрока и рейкастинг для взаимодействия с блоками.
*   **Resource Management:** Система автоматизированной загрузки текстур и аудио-ассетов, адаптированная под файловую систему Android (чтение напрямую из APK).

### 3. Ключевые реализованные технологии

#### 3.1. Оптимизированный рендеринг вокселей
Для достижения высокой частоты кадров (FPS) реализован алгоритм **Greedy Meshing**. Вместо отрисовки каждой грани блока по отдельности, движок объединяет соседние грани с одинаковой текстурой в одну большую полигональную сетку. Это на порядок снижает количество вызовов отрисовки (Draw Calls) и нагрузку на GPU.

#### 3.2. Многопоточный World Streamer
Система стриминга мира работает в отдельных фоновых потоках. Пока игрок перемещается, движок заранее генерирует новые чанки и строит для них геометрию (меш), не блокируя основной поток отрисовки. Это исключает "фризы" и рывки при исследовании бесконечного мира.

#### 3.3. Система сохранений и сериализации
Реализован модуль **WorldSave**, обеспечивающий персистентность мира. 
*   Изменения, внесенные игроком (сломанные или поставленные блоки), сохраняются в бинарном формате.
*   Метаданные (Seed мира, положение игрока) хранятся в формате JSON для удобства отладки.

#### 3.4. Аудио-движок на базе SDL_AudioStream
Разработана собственная система декодирования OGG-файлов (библиотека `stb_vorbis`). Аудиоданные подаются в динамические потоки SDL3, что позволяет реализовать фоновую музыку и звуковые эффекты без использования тяжелых сторонних движков вроде FMOD или Wwise.

### 4. Особенности реализации под Android
Портирование на Android стало ключевым этапом проверки архитектуры на гибкость.
*   **Графика:** Использование Vulkan позволило избежать ограничений устаревшего OpenGL ES.
*   **Жизненный цикл:** Обработка событий `WILL_ENTER_BACKGROUND` и `DID_ENTER_FOREGROUND` для корректного освобождения и восстановления графических ресурсов.
*   **Текущая задача:** В процессе отладки находится корректная передача `SurfaceTransform` от Android к Vulkan Swapchain для исправления поворота изображения на 90 градусов.

### 5. Заключение и результаты
На данный момент проект представляет собой готовую технологическую базу для создания полноценной игры. Реализованы наиболее сложные части движка: многопоточный стриминг, низкоуровневый рендеринг и система кроссплатформенного взаимодействия с ресурсами. Проект демонстрирует возможность создания сложного ПО с использованием современного промышленного стандарта C++20.
